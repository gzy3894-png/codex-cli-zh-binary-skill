安装脚本:
https://raw.githubusercontent.com/gzy3894-png/codex-cli-zh-binary-skill/android-arm64-musl-installer/android-arm64-musl/install.sh


1. Termux 裸环境完整安装
-----------------------

刚装好的 Termux 推荐复制下面两行执行。第一行安装下载器和证书，
第二行执行安装脚本；安装脚本默认会继续安装 Python/Node.js/Git/SSH/
ripgrep/fd/jq/clang/make/cmake/ninja 等工作环境依赖。

DEBIAN_FRONTEND=noninteractive apt-get update && DEBIAN_FRONTEND=noninteractive apt-get -o Dpkg::Options::=--force-confdef -o Dpkg::Options::=--force-confold install -y ca-certificates curl
curl -fsSL https://raw.githubusercontent.com/gzy3894-png/codex-cli-zh-binary-skill/android-arm64-musl-installer/android-arm64-musl/install.sh | sh


2. 已经有 curl 的环境
--------------------

curl -fsSL https://raw.githubusercontent.com/gzy3894-png/codex-cli-zh-binary-skill/android-arm64-musl-installer/android-arm64-musl/install.sh | sh


3. 只有 wget 的环境
------------------

wget -O- https://raw.githubusercontent.com/gzy3894-png/codex-cli-zh-binary-skill/android-arm64-musl-installer/android-arm64-musl/install.sh | sh


4. 轻量依赖模式
---------------

只安装 Codex 运行所需的轻量依赖，不安装完整 Python/Node/native build 工作环境:

DEBIAN_FRONTEND=noninteractive apt-get update && DEBIAN_FRONTEND=noninteractive apt-get -o Dpkg::Options::=--force-confdef -o Dpkg::Options::=--force-confold install -y ca-certificates curl
curl -fsSL https://raw.githubusercontent.com/gzy3894-png/codex-cli-zh-binary-skill/android-arm64-musl-installer/android-arm64-musl/install.sh | CODEX_ZH_DEPS_PROFILE=minimal sh


5. 如果上次卡在 bash.bashrc 提问
--------------------------------

先执行下面两行修复 Termux 的 dpkg 未完成配置。它会自动保留你当前的
bash.bashrc，不再弹 Y/I/N/O/D/Z 提问:

DEBIAN_FRONTEND=noninteractive dpkg --force-confdef --force-confold --configure -a
DEBIAN_FRONTEND=noninteractive apt-get -o Dpkg::Options::=--force-confdef -o Dpkg::Options::=--force-confold -f install -y

7. 安装结果
-----------

默认二进制安装到:
$PREFIX/bin/codex-zh-bin

包装命令安装到:
$PREFIX/bin/codex-zh

并把命令 codex 指向 codex-zh。

包装命令会自动修正 HOME、CODEX_HOME、TMPDIR 和 PATH，避免 root/su
shell 下 Codex 把状态库写到 /.codex。

安装后检查:
codex --version


8. su/root shell 里 codex 找不到
-------------------------------

如果提示:
/system/bin/sh: codex: inaccessible or not found

先执行:
export PATH="/data/data/com.termux/files/usr/bin:$PATH"

再执行:
codex --version
codex

如果之前已经出现:
Location: /.codex/state_5.sqlite

重新运行安装脚本即可。新版包装命令会把 HOME 固定回:
/data/data/com.termux/files/home

注意:
Termux 禁止 root/su shell 运行 apt/pkg。新版安装脚本在 root/su 下会跳过
依赖安装，只修复/安装 codex 包装命令。要补装 Python/Node/Git 等依赖，
请先执行 exit 回普通 Termux 用户，再运行完整安装命令。
